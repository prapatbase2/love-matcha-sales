# Love Matcha Sales v1.4.0

## สิ่งที่เพิ่ม/แก้ใน v1.4.0

### Dashboard
- เพิ่มตัวกรองดู “เดือนนี้” หรือเลือกช่วงวันที่ย้อนหลังเองได้
- เลือกแสดงรวมทุกสาขา หรือเลือกเฉพาะบางสาขาได้
- เพิ่ม KPI: รายได้รวม, รายจ่ายรวมจากหน้ายอดขาย (ค่านม + รายจ่ายอื่น), รายจ่ายเจ้าของลง, เอาเงินสดให้เจ้าของ, จำนวนแก้วที่ใช้, รายได้หลังหักค่าใช้จ่าย
- รายการล่าสุดแสดง: วันที่, สาขา, สถานะ, รายได้รวมทั้งหมด, รายจ่ายรวมทั้งหมด, เอาเงินสดให้เจ้าของ, เงินสดขาด/เกิน, หมายเหตุ

### ยอดขาย
- หากเมื่อวานยังไม่ได้ลงข้อมูลหรือเลือกหยุดร้านของสาขาที่มองเห็น ระบบจะแจ้งเตือนให้ย้อนกลับไปลงข้อมูล/เลือกหยุดร้าน

### เช็คชื่อ
- ตารางเช็คชื่อเรียงจากวันที่ 1 ไปปลายเดือน
- หากข้ามวันในเดือนนั้นแล้วยังไม่ได้เช็คชื่อ ระบบจะแสดงวันที่ที่ยังไม่ได้ลง
- ผู้ใช้ที่อยู่สาขาหนองคายมีช่องติ๊ก “พาร์ทไทม์ Rendo” และเลือกเวลา 18:00-22:00 ได้ทีละครึ่งชั่วโมง
- แสดงตารางลงเวลา Part-time Rendo แยกใต้ตารางเช็คชื่อ เมื่อมีรายการ

### ค่าตอบแทน
- เพิ่ม OT พาร์ทไทม์ Rendo สำหรับพนักงานสาขาหนองคาย
- เจ้าของ/ผู้จัดการตั้งค่าค่าตอบแทน Part-time Rendo บาท/ชั่วโมงได้ในหน้าค่าตอบแทน
- การคำนวณ OT ทำขนมในหน้าค่าตอบแทนจะอ้างอิงชื่อขนม/ราคาขาย/% OT ปัจจุบันจากการตั้งค่า เพื่อให้ปรับตามจริงเมื่อมีการเปลี่ยนแปลง

### ลงรายจ่าย
- เพิ่มหน้า “ลงรายจ่าย” ถัดจากเช็คชื่อ ก่อนเบิกเงิน เห็นเฉพาะเจ้าของและผู้จัดการ
- รองรับรายจ่ายประจำแบบรวมทุกสาขาแล้วกระจายตาม % ยอดขาย หรือเฉพาะสาขา
- รองรับรายจ่ายอื่นรายวันแบบรวมทุกสาขาหรือเฉพาะสาขา
- ดึงรายจ่ายค่าตอบแทนจากหน้าค่าตอบแทนที่บันทึกไว้แล้ว โดยใช้ “ต้นทุนรวม+ปกส.” / “ต้นทุนรวม”
- แสดงสรุปรายจ่ายเจ้าของลงรวมและภาระรายจ่ายของแต่ละสาขาในเดือนนั้น

### ระบบ
- ประวัติระบบไม่แสดง/เน้นรายการสำรองข้อมูลในหน้า “ระบบ” แล้ว
- อัปเดตเลขเวอร์ชันและ Service Worker cache เป็น v1.4.0

## วิธีอัปเดต
1. แตก ZIP นี้
2. อัปโหลดไฟล์ทั้งหมดทับของเดิมใน GitHub Pages repository
3. รอ GitHub Pages deploy เสร็จ
4. เปิดเว็บแล้ว refresh 1-2 ครั้ง หรือปิด/เปิดแอปบนมือถือใหม่ เพื่อให้ Service Worker โหลด v1.4.0

## หมายเหตุสำคัญ
- หน้า “ลงรายจ่าย” ดึงค่าตอบแทนอัตโนมัติจากข้อมูลที่บันทึกในหน้า “ค่าตอบแทน” แล้ว หากยอดเงินเดือน/รายวันยังไม่ขึ้น ให้กลับไปกดบันทึกแถวพนักงานในหน้าค่าตอบแทนก่อน
- รายจ่ายรวมทุกสาขาจะกระจายตามสัดส่วนยอดขายแต่ละสาขาในเดือนนั้น หากเดือนนั้นยังไม่มียอดขาย ระบบจะแบ่งเท่ากันชั่วคราว
