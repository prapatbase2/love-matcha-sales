# คู่มือติดตั้ง Love Matcha Sales v1.0


## อัปเดตแก้ไขล่าสุด

เวอร์ชัน ZIP นี้แก้ตามที่แจ้งแล้ว:

- ใส่ Firebase config ของโปรเจกต์ `love-matcha-sale-3de45` ให้แล้วใน `index.html`
- แก้หน้ายอดขาย: เมื่อเปลี่ยนสาขาหรือวันที่ ระบบจะล้างฟอร์มก่อนโหลดข้อมูลของสาขานั้น ไม่เอายอดเดิมค้างไปทุกสาขา
- หน้า OT ทำขนมในหน้ายอดขาย เหลือให้เลือกแค่ชนิดขนม / จำนวน / หมายเหตุ ไม่โชว์ราคาและเปอร์เซ็นต์
- เพิ่มการตั้งค่าชนิดขนม / ราคา / เปอร์เซ็นต์ OT ในหน้า “ค่าตอบแทน” สำหรับเจ้าของและผู้จัดการ
- โบนัสรายวันและโบนัสรายเดือนในหน้าค่าตอบแทน คำนวณอัตโนมัติจากยอดขายตามกติกาที่กำหนด
- เพิ่มปุ่ม “สรุป/แชร์” ในหน้าค่าตอบแทน เพื่อแชร์ข้อความสรุปให้พนักงานทาง LINE หรือคัดลอกข้อความ
- ปรับหน้าประวัติให้ไม่แสดงรายการรก เช่น ซ่อน/แสดงประวัติ และเปลี่ยนรายละเอียดจากโค้ด JSON เป็นข้อความอ่านง่าย
- เพิ่ม Offline lock: ตอนออฟไลน์เปิดดูได้ แต่ช่องกรอก/ปุ่มบันทึก/แก้ไข/ลบ/Restore จะถูกปิด และขึ้นเตือน “ออฟไลน์อยู่ กรุณาต่ออินเตอร์เน็ตก่อน”

## สำคัญมาก: ห้ามเปิด index.html ด้วยการดับเบิลคลิกโดยตรง

เว็บนี้ใช้ Firebase, JavaScript Module และ PWA Service Worker จึงต้องเปิดผ่าน `https://` เช่น GitHub Pages หรือผ่าน `http://localhost` เท่านั้น

ถ้าเปิดแบบดับเบิลคลิกเป็น `file://.../index.html` เบราว์เซอร์อาจบล็อกไฟล์ JavaScript และจะค้างที่ “กำลังโหลดระบบ...”

วิธีทดสอบในเครื่อง Windows:

1. แตก ZIP ออกมาก่อน
2. ดับเบิลคลิกไฟล์ `เปิดทดสอบในเครื่อง.bat`
3. รอให้เบราว์เซอร์เปิด `http://localhost:8000`
4. ถ้าเครื่องไม่มี Python ให้ใช้วิธีอัปโหลดขึ้น GitHub Pages แทน

หมายเหตุ: ใช้งานจริงหลายเครื่องพร้อมกันควรใช้ GitHub Pages หรือโฮสต์ที่เป็น HTTPS ไม่ใช่เปิดไฟล์จากเครื่องโดยตรง


เว็บแอปนี้เป็นระบบบันทึกยอดขายรายวัน/รายเดือน เช็คชื่อ เงินเบิกล่วงหน้า ค่าตอบแทน ประวัติ สำรองข้อมูล และ Restore สำหรับร้าน Love Matcha

ออกแบบจากแนวคิดไฟล์ Excel ที่แนบมา:
- มีชีตลงชื่อทำงาน
- มีชีตยอดขายแยกสาขา เช่น หนองคาย / อุดรธานี / ศรีราชา / นครพนม / ออกบูธ
- คอลัมน์หลัก เช่น วัน/เดือน/ปี, รายชื่อเพื่อนร่วมกะ, จำนวนคน, รายได้รวม, Line Man, Grab, ยอดขายเฉลี่ยต่อคน, เงินเปิดกะ, เงินปิดกะ, ค่านม, รายจ่ายอื่น ๆ, นับจำนวนแก้ว

## ไฟล์ในชุดนี้

# 2) ตั้งค่า Firebase

## 2.1 สร้าง Firebase Project

1. เข้า Firebase Console
2. กด Add project / เพิ่มโปรเจกต์
3. ตั้งชื่อ เช่น `love-matcha-sales`
4. Google Analytics จะเปิดหรือปิดก็ได้
5. กด Create project

## 2.2 เปิด Anonymous Auth

1. ไปที่ Build > Authentication
2. กด Get started
3. ไปแท็บ Sign-in method
4. เลือก Anonymous
5. กด Enable
6. Save

## 2.3 เปิด Firestore Database

1. ไปที่ Build > Firestore Database
2. กด Create database
3. เลือก Production mode
4. เลือก location ใกล้ไทย เช่น asia-southeast1 หรือ asia-east2 ถ้ามี
5. กด Enable

## 2.4 ใส่ Firestore Rules

1. ไปที่ Firestore Database > Rules
2. เปิดไฟล์ `firebase-rules.txt`
3. Copy ทั้งหมดไปวางทับ Rules เดิม
4. กด Publish

Rules ชุดนี้บังคับว่าต้องผ่าน Firebase Anonymous Auth ก่อน ไม่ใช่ฐานข้อมูลสาธารณะ

ข้อควรระวัง:
- เพราะเว็บนี้เป็น static web บน GitHub Pages และใช้ PIN 4 ตัวภายในร้าน
- Firebase Rules ไม่สามารถตรวจ PIN ได้โดยตรง
- สิทธิ์ เจ้าของ/ผู้จัดการ/หัวหน้างาน/พนักงาน ถูกควบคุมในหน้าเว็บ
- ถ้าต้องการความปลอดภัยสูงมาก ควรเพิ่มระบบ server / Cloud Functions / Custom Claims ในอนาคต

## 2.5 เอา Firebase config มาใส่ใน index.html

1. ไปที่ Project settings
2. เลื่อนลงมาที่ Your apps
3. กดไอคอน Web `</>`
4. ตั้งชื่อ app เช่น `love-matcha-web`
5. กด Register app
6. ระบบจะแสดง firebaseConfig ประมาณนี้

```js
const firebaseConfig = {
  apiKey: "...",
  authDomain: "...",
  projectId: "...",
  storageBucket: "...",
  messagingSenderId: "...",
  appId: "..."
};
```

7. เปิดไฟล์ `index.html`
8. หาโค้ดนี้

```js
window.LOVE_MATCHA_FIREBASE_CONFIG = {
  apiKey: "ใส่ของคุณ",
  authDomain: "ใส่ของคุณ",
  projectId: "ใส่ของคุณ",
  storageBucket: "ใส่ของคุณ",
  messagingSenderId: "ใส่ของคุณ",
  appId: "ใส่ของคุณ"
};
```

9. แก้ค่า `"ใส่ของคุณ"` ให้เป็นค่าจริงของ Firebase
10. Save

# 6) ตั้งค่า Google Apps Script Backup ไป Google Drive

## 6.1 สร้าง Apps Script

1. เข้า https://script.google.com
2. กด New project
3. ลบโค้ดเดิมใน `Code.gs`
4. เปิดไฟล์ `apps-script-backup.gs`
5. Copy โค้ดทั้งหมดไปวาง
6. กด Save
7. ตั้งชื่อ project เช่น `Love Matcha Backup`

## 6.2 Deploy เป็น Web App

1. กด Deploy
2. เลือก New deployment
3. กดรูปเฟือง เลือก Web app
4. Description ใส่ `Love Matcha Backup`
5. Execute as เลือก `Me`
6. Who has access เลือก `Anyone`
7. กด Deploy
8. ถ้ามีหน้าขอสิทธิ์ ให้กด Authorize access
9. Copy Web app URL ที่ลงท้ายด้วย `/exec`

## 6.3 ใส่ URL ในเว็บ

1. เข้าเว็บ Love Matcha
2. Login เป็นเจ้าของ
3. เข้าเมนู “สำรอง”
4. วาง URL ในช่อง Google Apps Script Web App URL
5. เลือก Auto Backup:
   - ปิด
   - สำรองทุกกี่นาที
   - สำรองเมื่อมีการทำรายการ
   - ทั้งสองแบบ
6. กดบันทึกตั้งค่า
7. กด Backup ไป Drive ตอนนี้
8. ไป Google Drive จะเจอโฟลเดอร์ `LoveMatcha_Backups`

ชื่อไฟล์ backup จะประมาณ:

```text
LoveMatcha_Backup_2026-05-01_14-30-22.json
```

# 7) Offline mode

เว็บนี้เป็น PWA และมี service worker

เมื่อเคยเปิดเว็บแล้ว:
- ไฟล์หน้าเว็บจะถูก cache
- Firestore จะพยายามเปิดข้อมูลล่าสุดจาก cache ได้
- ถ้า offline ระบบจะแสดงคำเตือน “ออฟไลน์อยู่ กรุณาต่ออินเตอร์เน็ตก่อน”
- ปุ่มบันทึก / แก้ไข / ลบ / Restore จะถูกปิด
- ใช้ดูข้อมูลล่าสุดที่เคยโหลดไว้ได้เท่านั้น

ห้ามบันทึกข้อมูลตอน offline เพื่อป้องกันข้อมูลชนกันหรือเสียหาย

# 9) Data model ใน Firestore

ระบบออกแบบ collection ไว้ดังนี้:

- `users` ผู้ใช้
- `branches` สาขา
- `dailySales` ยอดขายรายวัน ใช้ document id แบบ `branchId_date`
- `dailyExpenses` รายจ่ายอื่น ๆ แยกรายการ
- `cupCounts` ข้อมูลแก้ว
- `dessertOT` ข้อมูล OT ทำขนม
- `attendance` เช็คชื่อ
- `salaryAdvances` เงินเบิกล่วงหน้า
- `compensationSettings` เผื่อขยายในอนาคต
- `compensationRecords` ค่าตอบแทนรายเดือน
- `appSettings` ตั้งค่าระบบ
- `auditLogs` ประวัติการทำรายการ
- `backupsMetadata` metadata ของ backup

หลักการวันที่:
- วันที่เก็บในระบบเป็น ISO เช่น `2026-05-01`
- เดือนเก็บเป็น `monthKey` เช่น `2026-05`
- หน้าเว็บแสดงเป็น พ.ศ. เช่น `วันจันทร์ 1/5/2569`

# 10) สูตรคำนวณที่ใช้

## รายได้

```text
รายได้รวม = ยอดขายก่อนส่วนลด - ส่วนลด
รายได้รวมทั้งหมด = รายได้รวม + Line Man + Grab
ยอดขายเฉลี่ยต่อคน = รายได้รวมทั้งหมด / จำนวนคนทำงาน
```

## เงินสด

```text
เงินสดที่ควรเหลือ =
เงินสดเปิดกะ
+ เงินสดที่ขายได้ในวันนั้น
- ค่านม
- รายจ่ายอื่น ๆ
- เอาเงินสดออกให้เจ้าของ

เงินสดขาด/เกิน = เงินสดปิดกะ - เงินสดที่ควรเหลือ
```

ถ้าเงินสดขาด/เกินไม่เท่ากับ 0 ต้องกรอกสาเหตุ

## แก้ว

```text
แก้วที่ใช้จริง =
แก้วคงเหลือวันก่อน
+ แก้วที่เพิ่มวันนี้
- แก้วคงเหลือวันนี้
```

ถ้าติดลบต้องกรอกหมายเหตุ

## OT ทำขนม

```text
ค่าตอบแทน OT ทำขนม =
ราคาขาย * จำนวน * เปอร์เซ็นต์ / จำนวนคนที่ทำ
```

## ประกันสังคม

```text
ประกันสังคมฝ่ายลูกจ้าง = 5% ของเงินเดือน
ประกันสังคมฝ่ายนายจ้าง = 5% ของเงินเดือน
```

## ยอดโอนปลายเดือน

```text
ยอดที่ต้องโอนปลายเดือน =
เงินเดือน
+ ค่าตอบแทนเพิ่มเติม
+ OT ทำขนม
+ เงินเพิ่มออกบูธ
+ โบนัสรายวัน
+ โบนัสรายเดือน
- หักเงิน
- เงินเบิกล่วงหน้า
- ประกันสังคมฝ่ายลูกจ้าง
```

# 11) Restore Backup

Restore ทำได้เฉพาะเจ้าของ

รองรับ:
- JSON backup จากระบบนี้
- CSV ยอดขายที่ export จากระบบนี้

ขั้นตอน:
1. เข้าเมนู “สำรอง”
2. เลือกไฟล์ JSON หรือ CSV
3. ระบบจะแสดง Preview จำนวน records
4. กรอก PIN เจ้าของ
5. กด Restore
6. ระบบจะสร้าง backup ปัจจุบันก่อน restore ให้อัตโนมัติ
7. ยืนยัน restore

ข้อควรระวัง:
- Restore อาจเขียนทับข้อมูลเดิมที่ document id ซ้ำกัน
- ควรทดสอบกับ Firebase project สำรองก่อน ถ้าข้อมูลสำคัญมาก

# 12) ข้อควรระวังเรื่อง PIN

ระบบนี้ออกแบบตาม requirement ที่เจ้าของต้องเห็น PIN ได้ทั้งหมด จึงเก็บ PIN เป็นตัวเลขใน Firestore

ข้อดี:
- เจ้าของดู PIN ให้พนักงานได้
- Login ใช้งานง่าย ไม่ต้องใช้ email
- เหมาะกับร้านที่ใช้งานภายใน

ข้อเสีย:
- ไม่ปลอดภัยเท่าการ hash password
- ผู้ที่มีสิทธิ์เข้าถึง Firestore โดยตรงอาจเห็น PIN
- ไม่ควรใช้ PIN เดียวกับบัญชีธนาคารหรือบัญชีส่วนตัวอื่น

คำแนะนำ:
- ใช้ PIN เฉพาะระบบร้านนี้เท่านั้น
- จำกัดคนที่รู้ URL เว็บ
- อย่าเผยแพร่ Firebase config และ URL แบบไม่จำเป็น
- ถ้าในอนาคตต้องการความปลอดภัยสูง ให้เปลี่ยนเป็น Firebase Email Auth หรือ Custom Auth

- หน้าสรุปรายเดือน: แยกตารางรายวันตามสาขา เรียงวันที่ 1 ไปปลายเดือน และเปิด/ซ่อนรายละเอียดรายวันได้
- หน้าค่าตอบแทน: ย้ายช่องกำหนดค่าตอบแทนพนักงานรายวันไปไว้หลังตารางค่าตอบแทนรายวัน และใช้รายชั่วโมงแบบบาท/ชั่วโมง
- เงินเดือนพนักงานทั่วไป: ถ้าเดือนนี้ยังไม่เคยบันทึก ระบบจะลอกจากเดือนก่อนหน้าให้อัตโนมัติ แต่ยังแก้ได้
- Save / ส่งยอด: Auto Backup แบบเมื่อมีรายการจะทำงานเบื้องหลัง ไม่รอให้ Backup จบก่อน จึงลดอาการหน่วง
- Backup / Restore: เหลือเฉพาะ JSON ยกเลิก Google Sheet / Excel
